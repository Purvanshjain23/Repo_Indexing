F001: BAIZSF - Business Logic Function

Priority: Medium GENERIC - AI Enhancement Needed

1. Parameters and Business Data Elements

The BAIZSF function initializes the subfile and validates user warehouse access with these critical data elements:

  • P1TAST (First Time Flag): Determines if this is initial program load. When 'Y', initializes company number (#2AIC3), warehouse code (#CCPCD), and scheduled ship date (#2AODT) from parameter fields.
  • #CCPCD (Warehouse Code): User-entered warehouse identifier validated for access authorization. Blank value triggers "Warehouse Code Required" error message.
  • ##USR (User Profile): Current user ID used to retrieve warehouse access permissions from security files.
  • WN0001-WN0021 (User Warehouse Access Flags): Permission indicators for Inventory Transactions, Lot/Loc Tracking, Shipping Transactions, and Cross Applications retrieved via SDRVGN subroutine.
  • ##RRMX (Subfile Record Maximum): Counter reset to zero to clear previous subfile content before reload.

2. Business Logic Summary

  • Primary Business Workflow: Clears subfile (SETON indicator 80), retrieves company/warehouse descriptions via SARVGN and SBRVGN, validates warehouse code is entered, retrieves user profile via SCRVGN, obtains warehouse access permissions via SDRVGN, and validates user has required access level.
  • Key Decision Points: P1TAST='Y' (first time initialization), #CCPCD=*BLANK (warehouse code validation), WN0020=*BLANK (user code model check), WN0021='N' (access denied check), and multiple access area validations ('I' for Inventory, 'S' for Shipping, 'C' for Cross Applications).
  • Exception Management: Sends USR2015 message when warehouse code is blank and exits to BAEXIT. Sends USR2776 access denied message when user lacks warehouse permissions and exits to BAEXIT. Sends USR2792 message for specific functional area access denials.

3. Detailed Logic Explanation and Business Process Flow

BAIZSF begins by clearing the subfile using indicator 80, writing the subfile control record, then turning indicator 80 off. If P1TAST flag is 'Y' (first time), the function initializes control fields from parameter list including company number, warehouse code, and scheduled ship date. The function calls SARVGN to retrieve company name and SBRVGN to retrieve warehouse description for display. Critical warehouse code validation occurs next - if #CCPCD is blank, message USR2015 displays and processing terminates via GOTO BAEXIT. Upon passing validation, the function initializes 21 user warehouse access work fields (WN0001-WN0021) to blank. SCRVGN retrieves the user profile model code (WN0020). Based on whether WN0020 is blank, the function calls SDRVGN passing either ##USR (current user) or WN0020 (model user) along with warehouse code to retrieve permission flags. These flags indicate access levels (N=None, R=Read, U=Update) for Inventory, Shipping, and Cross Application functional areas. If WN0021='N' (record not found), access denied message USR2776 displays and processing exits. The function then validates specific access area requirements, constructing and sending area-specific denial messages via CARVC2R service when permission conflicts exist.

F002: BBLDSF - Business Logic Function

Priority: Medium GENERIC - AI Enhancement Needed

1. Parameters and Business Data Elements

BBLDSF loads subfile pages with extensive filtering using these business data elements:

  • *IN82 (End of File Indicator): Controls DOWEQ loop - processing continues while '0', terminates when '1' (EOF reached).
  • ##RROK (Records OK Counter): Tracks successfully loaded subfile records. Compared against ##SFLN (subfile lines needed) to determine when page is full.
  • ##RRRD (Records Read Counter): Tracks total database records examined. Compared against W0SLM (scan limit maximum) to prevent excessive reads.
  • #2T4NB, #2VDST, #2RPNB (Selection Criteria): Schedule sequence number, load type, and total orders filters entered by user on screen header.
  • #2AODT, #2KZDT, #2AXTM, #2RMNB (Positioning Keys): Scheduled ship date, deadline departure date/time, and load ID used for database positioning and filtering.
  • #CHGCD (Item Code): Optional item filter - when non-zero, only loads containing this item are displayed via SHRVGN validation.
  • #2BZNA (Carrier Code): Carrier filter with special '*' value to select loads with blank carriers.
  • W0RSL (Record Selected Flag): Tracks whether current database record passes all filters ('Y'=selected, 'N'=rejected).

2. Business Logic Summary

  • Primary Business Workflow: Re-establishes database position via READP/READ, calculates pages needed based on cursor position (W0RR0) or requested pages (W0SPG), enters loop reading load header records from @FJCPVB file, applies cascading selection filters (schedule, type, orders, dates, ID), validates company match, checks load status (E/C/L/U), validates item exists on load via SHRVGN, validates carrier code (including special '*' blank carrier logic), converts fields via MNCV#1, writes qualifying records to subfile.
  • Key Decision Points: W0RR0>0 (cursor positioning active), ##SFLN>999 (limit to 999 records), *IN82='0' AND ##RROK<##SFLN AND ##RRRD
  • Exception Management: CAB (Compare and Branch) operations jump to BB020 label when selection criteria fail, skipping processing of non-qualifying records. GOTO BBEXIT when no item filter (#CHGCD=0). Scan limit check prevents runaway reads by exiting loop when ##RRRD>=W0SLM. Displays Y2U0017 scan limit message and Y2U0008 no data message via calling functions.

3. Detailed Logic Explanation and Business Process Flow

BBLDSF begins by re-establishing database position using conditional READP (read previous) and READ operations to ensure proper file cursor placement. The function calculates required subfile pages by dividing cursor position (W0RR0) by page size (##SFPG) or using requested pages (W0SPG), then multiplies by page size to get total lines needed (##SFLN), capping at 999. The core processing loop executes while EOF indicator (*IN82) is '0', records loaded (##RROK) is less than lines needed (##SFLN), and records examined (##RRRD) is below scan limit (W0SLM). For each load header record read, the function applies selection filters in sequence: if schedule sequence (#2T4NB) is entered and database value (FJT4NB) is less than filter, CAB branches to BB020. Load type, order count, dates, and load ID filters follow similar pattern. The function calls MBFL#1 to map database fields to subfile work area, then sets W0RSL to 'Y' (selected). Company filtering activates when #2AIC3 is non-zero - if database company (FJAIC3) doesn't match and multi-company flag (FJRVSX) is 'N', W0RSL becomes 'N'. Load status must be E (Entered), C (Combined), L (Locked), or U (Unlocked) - other statuses trigger rejection. When item code (#CHGCD) is entered, function exits to BBEXIT if zero, otherwise calls SHRVGN to validate item exists on load detail - if YL0001 is not 'Y', W0RSL becomes 'N'. Carrier validation checks if #2BZNA is entered: special '*' value selects loads with blank carriers (rejecting those with assigned carriers), otherwise exact carrier code match is required. Schedule ship date validation ensures #2AODT matches FJAODT when date filter is active. Qualifying records (W0RSL='Y') have fields converted via MNCV#1, screen indicators set via GADSA1, then written to subfile, incrementing ##RR and ##RROK counters. Function exits loop when page full, scan limit reached, or EOF encountered.
