      *****************  RPG PROGRAM HEADING  ************************
      *
      * SYSTEM:      HOG PRODUCTION
      * PROGRAM:     HP256 - RETRIEVAL OF FARM SITE NAME ASSOCIATED
      *                      WITH A FARM SITE CODE
      * PROGRAMMER:  LEANNE FEDOR
      * CREATED:     01/06/96
      *
      * FUNCTION: THIS PROGRAM WAS REQUESTED AS AN INTERFACE FROM
      *           THE HOG PROCUREMENT AND EVALUATION SYSTEM TO THE
      *           HOG PRODUCTION SYSTEM
      *
      *           ITS PURPOSE IS TO RETRIEVE AND RETURN TO HPES THE
      *           NAME OF THE FARM SITE ASSOCIATED WITH THE NUMERIC
      *           FARM SITE CODE.
      *           THIS PROGRAM IS CALLED BY A SYNON PROGRAM IN HPES.
      *
      *           A RETURN CODE IS INCLUDED AS A PARM BUT NEVER USED IN
      *           THIS PROGRAM.
      *
      ****************************************************************
      * MODIFICATIONS:
      ****************************************************************
      * DATE      PROGRAMMER
      *
      * 04/13/98  LeAnne Fedor
      *
      * 06/05/01  LeAnne Fedor
      *           Recompile only. New field 'multisite' added to Farm Site file.
      *
      * 06/28/01  LeAnne Fedor
      *           Recompile only.
      *           Three fields (manager/supervisor/multisite) renamed in Farm Site file.
      *
      * 09/04/01  LeAnne Fedor
      *           Recompile only. New 'square feet' fields added to Farm Site file.
      *
      * 10/15/13  LeAnne Ramsey (E2831)
      *           Recompile only. Added field 'MTech Reference'.
      *
      * 10/22/13  LeAnne Ramsey (E2831)
      *           We will now pass the MTech References back out
      *           to the calling Synon program.
      /EJECT
      ****************************************************************
      * FILE SPECIFICATIONS
      ****************************************************************
      *
     FHSP018    IF   E           K DISK
      *    FARM SITE                  (KEY: FSFSCD)
      *
      /EJECT
      ****************************************************************
      * INPUT SPECIFICATIONS
      ****************************************************************
      *
      *---------------------------------------------------------------
      * STANDARD PROGRAM STATUS DATA STRUCTURE
      *---------------------------------------------------------------
      *    EXTERNALLY DEFINED AS UTPGFR (RECORD FORMAT: PGMDSR)
     D PGMDS         ESDS                  EXTNAME(UTPGFR)
      *
     D                 DS
     D  DUMMYD                 1      1
      *  THE PRECEDING DUMMY DATA STRUCTURE IS ONLY TO KEEP THE I-SPEC
      *  INFO FROM RUNNING OVER INTO THE 'CALC SPECS' ON THE COMPILE
      *  LISTING.  IT SERVES NO OTHER PURPOSE AND CAN BE DELETED
      *  AT ANY TIME!!!
      /EJECT
      ****************************************************************
      * MAIN LINE
      ****************************************************************
      *
      *
      * RETRIEVE THE FARM SITE NAME
      *
     C     XXFSCD        CHAIN     HSP018                             92
     C     *IN92         IFEQ      *OFF                                         IF FARM HIT
      *
     C                   move      fsmtechref    xxfsref
      *
     C                   SELECT
     C     XXPPCD        WHENEQ    *BLANK
     C                   MOVEL(P)  FSFSNM        XXFSNM
      *
     C     XXPPCD        WHENNE    *BLANK
     C     XXPPCD        ANDEQ     FSPPCD
     C                   MOVEL(P)  FSFSNM        XXFSNM
     C                   ENDSL
      *
     C                   ENDIF                                                  IF FARM HIT
      *
      *---------------------------------------------------------------
      * EOF PROCESSING
      *---------------------------------------------------------------
      *
     C                   SETON                                        LR
      /EJECT
      *---------------------------------------------------------------
      * *INZSR - INITIALIZATION SUBROUTINE
      *---------------------------------------------------------------
      *
     C     *INZSR        BEGSR
      *
      * THE PARM 'RETURN' WILL ALWAYS BE BLANK.
      *
     C     *ENTRY        PLIST
      *
      * INPUT PARMS
     C                   PARM                    RETURN            7
     C                   PARM                    XXFSCD
     C                   PARM                    XXPPCD
      * OUTPUT PARM
     C                   PARM                    XXFSNM
     C                   PARM                    XXFSREF
      *
      *
     C     *LIKE         DEFINE    FSFSCD        XXFSCD
     C     *LIKE         DEFINE    FSPPCD        XXPPCD
     C     *LIKE         DEFINE    FSFSNM        XXFSNM
     C     *LIKE         DEFINE    FSMTECHREF    XXFSREF
      *
     C                   MOVE      *BLANK        RETURN
     C                   MOVE      *BLANK        XXFSNM
      *
     C                   ENDSR
      /EJECT
